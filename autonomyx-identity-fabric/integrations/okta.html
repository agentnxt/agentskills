Step 1 of 4
Create an OAuth 2.0 app in Okta
We'll use OAuth 2.0 client credentials to read user identities from your Okta org — no SSWS token needed.
ℹ️ This creates a machine-to-machine (service app) in Okta. No user login or redirect URI is needed.
1
Log into your Okta Admin Console at https://your-domain.okta.com/admin
2
Go to Applications → Applications → click Create App Integration
3
Select API Services (machine-to-machine). Name it Autonomyx Identity Fabric
4
In the app's Okta API Scopes tab, grant these scopes:

okta.users.read okta.groups.read okta.apps.read
5
Copy the Client ID and Client Secret from the General tab — you'll enter them in the next step.
⚠️ The Client Secret is shown only once. Copy it before leaving the page.
